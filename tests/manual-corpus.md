# Manual test corpus

Корпус для ручной проверки плагина в Figma. Каждый блок — отдельный текстовый узел: создаёшь узел, вставляешь содержимое **из тройных бэктиков**, запускаешь плагин (`Plugins → Typo Graf`) и сверяешь сводку с числом ниже.

**Если плагин не показывает ожидаемое поведение:** в Figma `Plugins → Manage plugins → Typo Graf → удалить → Import from manifest.json` (это сбрасывает кэш билда).

---

## 1. Русский — комплексный (ёфикация, инициалы, тире, NBSP перед единицами, диапазон дат, минус, тысячи, квантификаторы)

**Ожидаемых правок: 22**, язык: `ru`

```
Шел ребенок 20 кг по № 5 — и т.д. А. С. Пушкин жил в 1799-1837 гг. Учет составил -300 ₽, скидка -15 % на 1234567 единиц. См. https://example.com или foo@bar.com.
```

**Что должно произойти:**
- `Шел → Шёл`, `ребенок → ребёнок`, `Учет → Учёт` — ёфикация (safe-словарь eyo-kernel)
- `№ 5` → `№ NBSP 5`
- `20 кг`, `300 ₽`, `15 %` — NBSP между числом и единицей
- `1799-1837 гг.` → `1799 [WJ] — [WJ] 1837 NBSP гг.` — em-dash обёрнут в word joiner (U+2060), чтобы Figma не разорвала диапазон по em-dash. Весь `в 1799—1837 гг.` — единый чанк.
- `и т.д.` → `и NBSP т. NBSP д.` — composite abbreviation. **Между `д.` и `А.` (началом инициалов) — обычный пробел**, не NBSP: composite-хвост не клеится к следующему слову.
- `А. С. Пушкин` — NBSP между инициалами и фамилией
- `-300 ₽`, `-15 %` → `−300 ₽`, `−15 %` — Unicode minus в финансовом контексте
- `1234567` → `1 234 567` — группировка тысяч NBSP
- `См. https://...` и `или foo@bar.com` — **обычный пробел** между сокращением/проклитикой и URL/email (раньше был баг с NBSP)
- `https://...`, `foo@bar.com` — URL и email **нетронуты**

## 2. Русский — кавычки, апострофы, частицы, г-н, однобуквенное «я», case в «И/или»

**Ожидаемых правок: 17**, язык: `ru`

```
Г-н Иванов сказал: "если бы ты ведем учет вовремя, всё было бы иначе". И/или мы шли бы вместе. Сделал ли я задачу? До н.э. — это давно.
```

**Что должно произойти:**
- `Г-н` → `Г‑н NBSP Иванов` — неразрывный дефис (U+2011) + NBSP после
- `"…"` → `«…»` — ёлочки
- `ведем → ведём`, `учет → учёт` — ёфикация
- `если NBSP бы`, `было NBSP бы`, `шли NBSP бы`, `Сделал NBSP ли` — NBSP **только перед** частицей (энклитика тянется к слову слева; справа от частицы — обычный пробел, поэтому `иначе`, `вместе` могут переноситься).
- `до н.э.` → `до NBSP н. NBSP э.` — composite abbreviation
- `И/или` — **заглавная сохраняется** (раньше был баг с потерей регистра)
- `я задачу` → `я NBSP задачу` — однобуквенное `я` приклеивается к следующему слову (Лебедев §31)
- **`мы`, `ты`, `это` остаются как есть** (многобуквенные местоимения — самостоятельные слова, не клеим)
- **`вместе. Сделал` — обычный пробел, не NBSP** (хвост слова на `-те.` не считается аббревиатурой)

## 3. Английский — smart quotes, апострофы, honorifics, e.g., currency

**Ожидаемых правок: 24**, язык: `en`

```
Dr. Smith said "don't touch the lovers' books", e.g. don't break them. The cost is $1234.5 and 300 $. Speed 60 mph or 100 km/h for 10-12 km. Mr. Brown vs. Mrs. Davis — who's right?
```

**Что должно произойти:**
- `Dr.`, `Mr.`, `Mrs.`, `vs.` + NBSP перед именем (honorifics)
- `"…"` → `“…”` (smart quotes)
- `don't → don’t`, `lovers' → lovers’`, `who's → who’s`
- `e.g. for example` → `e. NBSP g. NBSP for example`
- `$1234.5 → $1,234.5` — US currency format
- `300 $` → `300 NBSP $` — trailing currency
- `60 mph`, `100 km/h`, `10-12 km` — NBSP перед единицами, en-dash в диапазоне

## 4. Математика

**Ожидаемых правок: 26**, язык: `en` (детектится по латинице)

```
Формула: x^2 + y^3 = z^10. log10(x) + sin(x) = lim(x→0). Дробь 1/2 + 3/4. Допуск +- 0.5, поток a*b*c. Стрелки: A -> B, C <- D, E => F, G <=> H, X --> Y. Греческие \alpha + \beta = \gamma. sqrt(2) ≈ 1.41, pi ≈ 3.14, inf.
```

**Что должно произойти:**
- `x^2 → x²`, `y^3 → y³`, `z^10 → z¹⁰`
- `log10(x) → log₁₀ x`, `sin(x) → sin x`, `lim(x→0) → limₓ→₀`
- `1/2 → ½`, `3/4 → ¾`
- `+- → ±`
- `a*b*c → a · b · c` (с цепочкой)
- Стрелки: `→`, `←`, `⇒`, `⇔`, `⟶`
- `\alpha → α`, `\beta → β`, `\gamma → γ`
- `sqrt(2) → √2`, `pi → π`, `inf → ∞`
- `Pi` (заглавная) **не конвертится** — может быть именем

## 5. Знаки © ® ™

**Ожидаемых правок: 7**, язык: `en`

```
Copyright (c) 2026, Acme Inc(r). Bar (tm) is a trademark. Also (C), (R), (TM) — все варианты.
```

**Что должно произойти:**
- `(c)`, `(C)` → `©`
- `(r)`, `(R)` → `®`
- `(tm)`, `(TM)` → `™`

## 6. Французский — гийеметы, NNBSP, апостроф, em-dash

**Ожидаемых правок: 11**, язык: `fr`

```
Il dit "bonjour !" à 20 % et 5 km avant midi : c'est l'arbre. Aujourd'hui — un jour spécial. Le budget est de 1234567 euros.
```

**Что должно произойти:**
- `"…"` → `« NNBSP … NNBSP »`
- Узкий NBSP перед `! ? : »`
- `5 km`, `20 %` — узкий NBSP
- `c'est → c’est`, `l'arbre → l’arbre`, `Aujourd'hui → Aujourd’hui` — smart liaison apostrophe
- `1234567 → 1 234 567` (узкие NBSP)

## 7. Немецкий — z.B., тысячи точкой, кавычки

**Ожидаемых правок: 14**, язык: `de`

```
Schöne Grüße aus Berlin. z.B. das ist gut, u.a. Äpfel und d.h. nichts Besonderes. Pi ist 3,14, Budget 1234567 Euro. Er sagte "Hallo" und ging. Speed 5 km, 20 %, n.Chr. 2026.
```

**Что должно произойти:**
- `"Hallo" → „Hallo“`
- `z.B. → z. NBSP B. NBSP …`, аналогично `u.a.`, `d.h.`, `n.Chr.`
- `1234567 → 1.234.567` — тысячи точкой (немецкий стандарт)
- `5 km`, `20 %` — NBSP
- `Pi` НЕ конвертится в π (заглавная)
- `3,14` — десятичная запятая нетронута

## 8. Испанский — paired ¿/¡, кавычки, единицы

**Ожидаемых правок: 7**, язык: `es`

```
Cómo estás? Muy bien! El precio es 5 km por 20 %. Hola. Cómo va el día? Él dijo "hola" en español.
```

**Что должно произойти:**
- `Cómo estás?` → `¿Cómo estás?` (добавлен открывающий вопросительный)
- `Muy bien!` → `¡Muy bien!`
- `Cómo va el día?` → `¿Cómo va el día?`
- `5 km`, `20 %` — NBSP
- `"hola" → "hola"` (smart quotes для es)
- Детект работает через `ñ` в "español" (unique-маркер)

## 9. Итальянский — артикли, предлоги, сокращения

**Ожидаемых правок: 14**, язык: `it` (детектится по `ò` в "Però")

```
Però è bellissimo a Roma. Il gatto di Sig. Rossi è con la casa di Dott. Verdi. Budget 1234567 euro per 5 km e 20 %.
```

**Что должно произойти:**
- `Il NBSP gatto`, `di NBSP Sig.`, `con NBSP la`, `di NBSP Dott.` — артикли/предлоги
- `Sig. Rossi → Sig. NBSP Rossi`, `Dott. Verdi → Dott. NBSP Verdi` — abbr + capitalized name
- `5 km`, `20 %` — NBSP
- `1 234 567` — NBSP-разделители

## 14. Бренды — должны остаться нетронутыми

**Ожидаемых правок: 6**, язык: `ru` (доминирующий)

```
Я открыл iPhone и зашёл на Booking.com. Установлю JavaScript и Node.js. iphone (lowercase) — это уже обычное слово. Figma + FigJam + Adobe XD работают. Заметка про PayPal и eBay.
```

**Что должно произойти:**
- `iPhone`, `Booking.com`, `JavaScript`, `Node.js`, `Figma`, `FigJam`, `Adobe XD`, `PayPal`, `eBay` — **полностью нетронуты** (защищены маской)
- `iphone` (строчная) — без защиты, обрабатывается как обычный текст

## 15. URL и email — должны пройти нетронутыми

**Ожидаемых правок: 4**, язык: `ru` (доминирующий)

```
Подробнее на https://example.com/foo-bar?x=1&y=2#section и в почте foo.bar+test@example.co.uk. Также см. www.test.org и user@another.com.
```

**Что должно произойти:**
- Все URL и email — **нетронуты целиком** (внутренние `-`, `.`, `?`, `&`, `+` сохранены)
- NBSP только в обычных местах: после `см.`, после `и` (короткий союз)

---

## 16. Mixed languages — dominant script wins

**Ожидаемых правок: 2**, языки: `en` (для первого предложения, доминирует латиница) и `ru` (для второго, доминирует кириллица)

```
It is awesome, but мир exists in Russia. И наоборот: Привет, hello, как дела?
```

**Что должно произойти:**
- Каждый текстовый узел детектится по доминирующему скрипту. Если разделить на 2 узла — получишь `en` и `ru` отдельно. В едином узле побеждает тот скрипт, символов которого больше; при равенстве — кириллица.

## 17. Edge cases — NFC, эмодзи, длинные тексты

**Ожидаемых правок: 12**, язык: `ru`

```
Эмодзи рядом 🌍 ребенок 20 кг 🎈 и семья 👨‍👩‍👧. Café (decomposed) → café (precomposed). Текст с двумя  пробелами  внутри. Слово , с пробелом перед запятой ! И что ?
```

**Что должно произойти:**
- Эмодзи `🌍`, `🎈`, `👨‍👩‍👧` (ZWJ-семейка) — **целые, не разорваны**
- `ребенок → ребёнок`, `20 NBSP кг` — обычные правила
- `Café` (decomposed `Cafe` + combining acute) → `Café` (precomposed, NFC)
- Двойные пробелы внутри → одинарный
- Пробелы перед `,`, `!`, `?` — удалены

## Что должно НЕ делаться

Если ты увидел что-то из списка ниже — это **баг, репортуй**:

- **NBSP внутри слова на `-те.`, `-п.`, `-д.`, `-ч.`, `-г.`** — например, `вместе. Сделал` НЕ должно становиться `вместе. NBSP Сделал`. После таких слов идёт обычный пробел.
- **NBSP после `гг.` или `вв.`** — `1991 гг. Учёт` → `1991 NBSP гг.` (NBSP только **перед**), `гг. Учёт` остаётся с обычным пробелом.
- **`те` (местоимение) → `т. е.`** — `те самые яблоки` НЕ должно становиться `т. е. самые яблоки`.
- **`Pi` (с заглавной) → `π`** — заглавное Pi не конвертится. Только строчное `pi`.
- **Изменение URL или email** — они должны быть полностью неприкосновенны (в т.ч. внутренние `-`, `.`, `?`).
- **Сломанные эмодзи (половинки surrogate pair)** — эмодзи должны оставаться целыми.
- **Ёфикация слов с омонимами** (например, `все` → `всё`) — мы не делаем такие замены: `все` может означать множ. число «весь», поэтому остаётся.

## Что НЕ контролирует плагин

- **Где Figma перенесёт строку.** Плагин может только **запретить** перенос в конкретной точке (через NBSP), но не может его **навязать**. Перенос между предложениями (например, `Сделал` на новой строке после точки) — это поведение Figma's text engine, не плагина.
- **Перенос внутри URL.** Figma может разорвать `https://example.com` на `https://` и `example.com` — это её алгоритм line-breaking. Мы оставили URL «прозрачным» специально, чтобы copy-paste давал валидный адрес.
