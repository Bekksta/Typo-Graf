# Typo Graf

Плагин для Figma, который приводит текст в макетах к типографическим нормам — расставляет неразрывные пробелы, кавычки, тире, степени и единицы. Работает на семи языках, сохраняет стили и не требует подключения к сети.

## Почему он

- **Легковесный.** Бандл — около 52 КБ (вместе с переводами на 7 языков). Никаких кастомных UI-фреймворков, тяжёлых ассетов и стартовых заставок.
- **Без задержек.** Открывается мгновенно: всё, что плагин показывает — прогресс-бар и кнопку «Отменить». Никаких настроек, никаких онбордингов.
- **Оффлайн.** Все правила, словари и логика — в бандле. `networkAccess: none` в манифесте. Ничего не уходит наружу.
- **Сохраняет стили.** Правки применяются через диапазоны (`deleteCharacters` / `insertCharacters`), а не перезаписью текста узла — `bold`, `italic`, шрифты, размеры и цвета остаются на месте.
- **Не трогает URL и email.** Они маскируются на время прохода правил и возвращаются обратно.
- **Один батч — одно undo.** Все изменения откатываются обычным `Ctrl/⌘+Z`.

## Что плагин делает

### Область действия

- Если в Figma что-то выделено — обрабатывает текстовые узлы внутри выделения (включая вложенные).
- Если ничего не выделено — обрабатывает все текстовые узлы текущей страницы.
- Лимит за один запуск — 2000 узлов. Длинные узлы (>5000 символов в одном непрерывном фрагменте) пропускаются и попадают в финальную сводку.
- Узлы обрабатываются батчами по 150 — между батчами UI обновляет прогресс.

### Язык

Определяется автоматически на уровне каждого текстового слоя (не страницы целиком) — по скрипту и характерным диакритикам:

| Язык | Маркеры |
|------|---------|
| `ru` | кириллица (фолбэк) |
| `uk` | `і ї є ґ` |
| `en` | латиница (фолбэк) |
| `fr` | `ç é â æ œ` и т. п. |
| `de` | `ä ö ü ß` |
| `es` | `ñ á é í ó ú ¿ ¡` |
| `bcs` | `č ć š đ ž` (латиница bs/hr/sr) |

Сербская кириллица обрабатывается как русский (по ТЗ).

## Правила

### Общие (все языки)

- `...` → `…` (U+2026), лишние пробелы вокруг чистятся.
- `10-12` → `10–12` (en dash для числовых диапазонов).
- Двойные пробелы → одинарный.
- `12 кг`, `20 %`, `300 ₽` → NBSP между числом и единицей/валютой/процентом.
- `12''` → `12″`, `12'` → `12′` (праймы).
- `45 deg` → `45°`.

### Математика

Работает на инлайн-выражениях, не лезет в LaTeX/MathML-блоки.

- **Степени:** `x^2 → x²`, `y^10 → y¹⁰`, `10^-3 → 10⁻³`, `e^(x) → eˣ`, `(a+b)^n → (a+b)ⁿ`.
- **Индексы:** `x_1 → x₁`, `log_10(x) → log₁₀ x`.
- **Умножение:** `a*b → a · b`, `a×b → a×b`, `a·b → a · b`. Жирный (`**...**`) не трогается.
- **Деление:** `a/b → a / b`, `1/2 → ½`, `(a+b)/(c+d)` без замены `/` на `÷`.
- **Сравнения:** `!= → ≠`, `<= → ≤`, `>= → ≥`, `= → =` (с пробелами), пробелы вокруг `≈ ≃`.
- **Знаки:** `+- → ±`, `-+ → ∓`.
- **Стрелки:** `-> → →`, `<- → ←`, `--> → ⟶`, `=> → ⇒`, `<=> → ⇔`.
- **Константы и функции:** `pi → π`, `sqrt(x) → √x`, `inf → ∞`, `\alpha → α` (и остальные греческие), `sin(x) → sin x`, `cos(x) → cos x`, `log10(x) → log₁₀ x`, `lim(x→0) → limₓ→₀`, `∑_(i=1)^n a_i → ∑ᵢ₌₁ⁿ aᵢ`, `vec(a) → a⃗`.

### Русский (`ru`)

- NBSP после коротких предлогов и союзов: `в дом`, `на улице`, `и т. п.`
- NBSP перед частицами `бы`, `ли`, `же`.
- Инициалы: `А. С. Пушкин` — NBSP между инициалами и фамилией.
- Аббревиатуры с точкой: `г. Москва`, `ул. Ленина`, `№ 8`, `§ 104`, `1981 г.`, `IV гл.` — NBSP после.
- Составные сокращения: `и т. д.`, `т. е.`, `до н. э.` — нормализованы вместе с NBSP.
- Дефисные сокращения без точки: `г-н`, `г-жа`, `д-р`, `р-н` — неразрывный дефис (U+2011) внутри и NBSP после.
- Кавычки-ёлочки `«…»`. Пунктуация прижимается к закрывающей.
- Тире: двойной дефис `--` и одиночный `-` между не-цифровыми токенами → `—`, схема `слово<NBSP>—<space>слово`.
- Ёфикация по белому списку: `ребёнок`, `всё`, `шёл`, `тёмный` и др. Список расширяется в `src/dict/yo-pairs.json`.

### Английский (`en`)

- Smart quotes: `"..."` → `“…”`, вложенные `'...'` → `‘…’`.
- Апострофы в словах: `don't` → `don’t`.
- Праймы: `12''` → `12″`, `12'` → `12′`.
- `--` → `—` без навязывания пробелов вокруг (по исходному стилю).
- Диапазоны: `10-12 km` → `10–12 km`.
- Валюты: `$1234.5` нормализуется в US-формат `$1,234.5`. Порядок `$ 300` / `300 $` сохраняется, NBSP добавляется.
- NBSP после служебных слов (`a`, `an`, `the`, `and`, `but`, `or`, `to`, `of`, `in`, `on`, `at`, `by`, `as`, `I`).
- NBSP перед единицами: `15 km`, `20 %`.

### Французский (`fr`)

- Гийеметы `« … »` с узкими NBSP (U+202F) внутри.
- Узкий NBSP перед `; : ? ! »`.
- Число + единица/валюта/процент → узкий NBSP.

### Украинский (`uk`)

- Кавычки `«…»`.
- NBSP после коротких предлогов: `в`, `у`, `з`, `із`, `й`, `та`, `а`, `і`.
- Узкий NBSP после `№`, `§`, `стор.`, `рис.`, `м.`.
- Число + единица/валюта/процент → узкий NBSP.

### Немецкий (`de`)

- Кавычки `„…"`. Если в тексте уже стоит `»…«` — не трогает.
- NBSP перед единицами и валютой.
- Десятичная запятая не трогается.

### Испанский (`es`)

- Кавычки `“…”` по умолчанию. Если уже стоит `«…»` — не трогает.
- NBSP перед единицами и валютой.
- `¿` и `¡` остаются как есть, искусственных пробелов вокруг `:;!?` не добавляется.

### BCS (`bs`/`hr`/`sr` латиница)

- Кавычки нормализуются по встреченному варианту (`«…»` или `„…"`). По умолчанию — `„…"`.
- NBSP перед единицами и валютой.

## Что плагин НЕ делает

- **Не трогает URL и email.** Перед прохождением правил маскируются равной длины placeholder'ами, после — восстанавливаются обратно.
- **Не ломает переносы строк.** `\n` сохраняется, склейка через перенос не делается.
- **Не лезет в LaTeX/MathML-блоки.** Только короткие inline-операнды вида `a^2`, `1/2`, `lim(x→0)`.
- **Не редактирует символьное выделение внутри узла.** Только текстовые узлы целиком.
- **Не подключает кастомные шрифты.** Если в узле используется шрифт, недоступный в Figma, такой узел пропускается и попадает в итоговый счётчик `пропущено по шрифту`.

## Интерфейс

Окно 320×160 px:

- Прогресс-бар.
- Текущая операция (`Сканирование…`, `Обработка…`, `Отмена…`).
- Счётчик обработанных узлов: `X / Y`.
- Кнопка «Отменить» — останавливает после текущего узла; всё, что уже применилось, остаётся (откатывается общим `Ctrl/⌘+Z`).

UI лейблы и финальный notify локализованы под язык Figma (см. ниже). Каждая статистика — отдельное предложение, подсказка про `Ctrl+Z` — на следующей строке:

```
Готово. Языки: русский, английский. Изменений: 142. Затронуто узлов: 38.
Ctrl/⌘+Z — отменить
```

## Локализация

Плагин определяет язык интерфейса Figma через `navigator.language` (читается в UI iframe и передаётся в plugin sandbox первым сообщением на старте). Поддержаны семь языков:

| Локаль | Соответствие `navigator.language` |
|--------|-----------------------------------|
| Русский | `ru-*` |
| English | `en-*` и любая нераспознанная локаль (фолбэк) |
| Français | `fr-*` |
| Українська | `uk-*` |
| Deutsch | `de-*` |
| Español | `es-*` |
| BCS (Bosnian / Croatian / Serbian latin) | `hr-*`, `sr-*`, `bs-*` |

Если UI не успел отдать локаль за 1500 мс — берётся английский. На локализацию завязаны: лейбл текущей операции, кнопка «Отменить», текст финального notify (включая названия определённых в узлах языков) и сообщения об ошибках.

## Установка (для разработки)

```bash
npm install
npm run build       # одноразовая сборка
npm run dev         # сборка в watch-режиме
```

Затем в Figma desktop → **Plugins → Development → Import plugin from manifest** → выбрать `manifest.json` в корне репозитория.

```
src/
├── main.ts              entrypoint: сбор узлов, батчинг, UI, applyReplacements
├── ui.html              минимальный UI: прогресс, Cancel, текущая операция
├── i18n.ts              переводы UI/notify, detectUILocale, langName
├── types.ts             Language, Replacement, Plan
├── lang/
│   ├── detect.ts        определение языка слоя
│   └── maps.ts          типографические константы (NBSP, NNBSP, EM_DASH...)
├── lib/                 константы и хелперы по языкам
├── rules/
│   ├── common.ts        общие правила (многоточие, диапазоны, единицы)
│   ├── math.ts          инлайн-математика
│   ├── ru.ts            русские правила
│   ├── en.ts            английские правила
│   ├── fr.ts, uk.ts, de.ts, es.ts, bcs.ts
│   └── yoPairs.ts       ёфикация по белому списку
├── text/
│   ├── mask.ts          length-preserving маскирование URL/email
│   ├── diff.ts          LCS-дифф + извлечение свободных сегментов
│   └── ranges.ts        применение Replacement[] к TextNode без потери стилей
└── dict/
    └── yo-pairs.json    словарь ёфикации
```

### Поток обработки одного узла

1. `detectLanguage(node.characters)` — определение языка слоя.
2. `maskSensitive` — URL и email заменяются на placeholder'ы той же длины из приватной плоскости Unicode (`+`). Индексы сохраняются.
3. `extractFreeSegments` — разбиение маскированного текста на участки **вне** масок.
4. Каждый сегмент изолированно проходит пайплайн `applyMath → applyCommonRules → applyXxxRules` до фикспоинта (макс. 3 итерации).
5. `diffLCS` (Uint32Array) считает минимальный список замен между сегментом «до» и «после».
6. Замены сдвигаются на абсолютные индексы узла и собираются в общий список.
7. Шрифты узла подгружаются, `applyReplacements` применяет замены справа-налево через `deleteCharacters` + `insertCharacters`. Стили сохраняются.

### Производительность

- LCS-таблица — `Uint32Array` размером `(n+1)·(m+1)`. Память — 4 байта на ячейку.
- Сегментный подход держит размеры таблиц маленькими — типичный сегмент короче 200 символов.
- Между батчами вызывается `setTimeout(0)`, чтобы UI успевал перерисоваться.

## Ограничения (известные)

- Селекция работает на уровне узлов, а не отдельных символов внутри узла (Figma API не даёт символьного выделения).
- Кнопка Undo changes отсутствует — Figma делает это нативно через `Ctrl/⌘+Z` всей undo-группой.
- Кликабельных примеров в финальном отчёте нет: отчёт показывается через `figma.notify`.
- Правила для `fr`, `uk`, `de`, `es`, `bcs` покрывают основную часть ТЗ, но не все edge-кейсы — это база для расширения.
