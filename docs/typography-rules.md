# Свод правил типографа

Этот файл описывает **источники**, **набор применяемых правил** и **сознательно игнорируемые правила**.

## 2. Что мы делаем

### 2.1. Общие правила (любой язык)

**NFC-нормализация на входе** — `é` (decomposed: `e` + U+0301) приводим к precomposed `é` (U+00E9). Без этого правила одинаковые на вид строки не сравнивались бы и не редактировались бы единообразно.
Источник: UAX #15.

**CRLF → LF** — Windows-окончания строк нормализуем в Unix-стиль, чтобы не плодить смешанные переносы и неконсистентные позиции.
Источник: общепринятая практика.

**Сиротные переносы в конце узла** — `\n` / `\r` (один или несколько) в самом конце текстового узла срезаются. Это мусор, остающийся после ручного редактирования в Figma: визуально пустая строка под текстом увеличивает высоту фрейма и сбивает auto-layout. Переносы **между** абзацами (`Абзац 1\n\nАбзац 2`) сохраняются — они осмысленные.
Источник: общепринятая практика.

**Многоточие** — три и более последовательных точек становятся `…` (U+2026). Лишние пробелы вокруг чистятся.
Источник: Розенталь §44, ГОСТ Р 7.0.97-2016.

**Знаки `(c)`, `(r)`, `(tm)`** — заменяются на `©`, `®`, `™` (любой регистр). Внутри URL/email/брендов не трогаются.
Источник: общепринятая практика.

**Число + единица измерения** — между ними неразрывный пробел (NBSP): `5 кг`, `20 %`, `300 ₽`. Список единиц курируемый: метрические (м, см, мм, км, кг, г, т, л, мл), время (мин, сек, ч), вычислительные (МБ, ГБ, ТБ, Гц, кГц, МГц, Вт, кВт, В, А, Ом), угловые (°, °C, °F), валюты (₽, €, $, £, ¥, R$, zł).
Источник: ГОСТ 8.417-2002 (метрические), общепринятая типографика.

**Группировка тысяч** — для чисел от 5 цифр расставляются неразрывные разделители тысяч. Локально: NBSP для ru/pl/it/pt/nl (`1 234 567`), узкий NBSP (NNBSP, U+202F) для uk/fr, точка для de (`1.234.567`). Для en — запятая только для валютных чисел с префиксом (`$1234.5` → `$1,234.5`). Для es группировка не выполняется.
Источник: ГОСТ 8.417-2002 (русский), CLDR (локальные форматы).

**Защита URL, email, брендов** — токены замещаются placeholder'ами равной длины в Unicode-области частного использования (U+E000–U+F8FF) до прохода правил и восстанавливаются после. Бренды берутся из `src/dict/brands.txt` (~50 названий: iPhone, JavaScript, Booking.com, Figma и др.), регистр имеет значение.
Источник: внутренняя архитектура.

### 2.2. Русский язык (`ru`)

**Проклитики → NBSP справа.** Слова, тянущие к себе следующее: короткие предлоги (в, к, с, у, о, на, по, до, от, из, за, об, во, ко, со, при, про, под, над, без), союзы (и, а, но, да, или, либо), отрицания (не, ни), эмфатические/указательные частицы (даже, только, лишь, вот, ведь, разве). Однобуквенное местоимение `я` — отдельным исключением, по правилу «однобуквенные слова не должны висеть в конце строки».
Источник: Лебедев §31, Мильчин §10.5.

**Энклитики → NBSP слева.** Постпозитивные частицы `бы`, `ли`, `же`, `ль` — тянутся к предыдущему слову. Пример: `всё было бы иначе` → `всё было<NBSP>бы иначе` (слово `иначе` свободно переносится).
Источник: Лебедев §32.

*Инвариант:* списки проклитик и энклитик не пересекаются — слово либо тянет вправо, либо влево.

**Инициалы.** `А. С. Пушкин` — NBSP между инициалами и между инициалами и фамилией. Работает и для двух частей (`А. С.`), и для трёх (`А. С. Пушкин`).
Источник: Мильчин §13.4.

**Сокращения с точкой.** Делятся на два класса:
- *Единицы* (`см`, `мм`, `м`, `км`, `г`, `кг`, `т`, `тыс`, `млн`, `млрд`, `мин`, `сек`, `ч`) — клеятся к **числу** через NBSP: `5 кг`, `1991 г.` Не клеятся к следующему слову, если перед сокращением была цифра.
- *Общие* (`г.` город, `ул.`, `№`, `стр.`, `рис.`, `табл.`, `п.`, `гл.`, `разд.`, `обл.`, `кв.`, `корп.` и т. д.) — клеятся к **следующему слову** через NBSP: `г. Москва`, `стр. 12`.

**Closing-сокращения `гг.` / `вв.`.** NBSP **только перед** (`1991<NBSP>гг.`), после — обычный пробел: эти сокращения замыкают число, перенос строки после них допустим.
Источник: Мильчин §13.5.

**Composite-сокращения.** Устойчивые сочетания нормализуются:
- `т.д.` / `т д` / `т.д` → `т. д.`
- `т.п.` → `т. п.`
- `т.е.` → `т. е.`
- `н.э.` → `н. э.`
- `до н.э.` → `до н. э.`
- `и др.`, `и пр.` → `и др.`, `и пр.`

Между буквами сокращения — NBSP, чтобы части не разрывались. **Хвост composite** (последняя буква, например `д.` в `и т. д.`) НЕ клеится к следующему слову через NBSP, иначе `и т. д. А. С. Пушкин` становился бы единым неразрывным чанком.
Источник: Мильчин §13.6, Лебедев §95.

**Дефисные сокращения без точки.** `г-н`, `г-жа`, `д-р`, `р-н`, `тов-щ`, `проф-а` и их падежные формы — внутри неразрывный дефис U+2011, после — NBSP перед именем: `г-н<NBH>Иванов` (где NBH = U+2011).
Источник: Мильчин §13.4.

**Кавычки.** «Ёлочки» `«…»` как основные. Прямые `"…"` нормализуются автоматически. Знаки препинания (точка, запятая, многоточие) затягиваются внутрь закрывающей кавычки.
Источник: ГОСТ Р 7.0.97-2016, Розенталь §54.

**Тире (em-dash).**
- `--` (двойной дефис) → `—`.
- `-` между нецифровыми токенами → `—`.
- Em-dash в середине фразы: NBSP слева, обычный пробел справа: `слово<NBSP>— слово`. Цифровые диапазоны не трогаются этим правилом (`1991—1995` остаётся слитным).
Источник: Розенталь §51, Лебедев §161.

**Диапазоны дат и месяцев.** Em-dash без пробелов, **обёрнутый Word Joiner'ом** (U+2060) с обеих сторон: `1991-1995` → `1991<WJ>—<WJ>1995`, `январь-март` → `январь<WJ>—<WJ>март`. Word Joiner запрещает Unicode-движкам разрывать строку по em-dash (без него Figma по UAX#14 рвёт `1991—1995` на `1991—` и `1995`).
Источник: Лебедев §161 (em-dash без пробелов в диапазонах), UAX #14 (для понимания обхода).

**Юникодный минус.** В финансовых/процентных/единичных контекстах ASCII-дефис заменяется на минус U+2212: `-300 ₽` → `−300 ₽`, `-15 %` → `−15 %`, `-2 °C` → `−2 °C`. Дефис в буллет-списках не трогается.
Источник: ISO 80000-2, общепринятая математическая типографика.

**Число + квантификаторное существительное → NBSP.** Курируемый список: формы валют (`рубль/рубля/рублей/копейка/доллар/евро/цент`), времени (`секунда/минута/час/день/неделя/месяц/год` во всех падежах), даты (родительный месяцев: `января/февраля/...`). Пример: `5 рублей`, `12 января`, `1991 году`, `300 минут`. Обычные счётные сущ. (`243 голубя`, `100 коробок`) НЕ клеятся.
Источник: Мильчин §13.5, Розенталь §136.

**Знаки `№`, `§`.** NBSP между ними и числом: `№<NBSP>5`, `§<NBSP>104`.
Источник: ГОСТ Р 7.0.97-2016.

**«И/или».** Нормализация устойчивого сочетания: `и / или` → `и/или` (без пробелов вокруг косой черты). Заглавная в начале предложения сохраняется: `И / или …` → `И/или …`.
Источник: общепринятое написание.

**Ёфикация.** Заменяются только формы из safe-словаря eyo-kernel (~107 000 словоформ). «Безопасные» — те, где нет омонима с `е` в той же словоформе. Поэтому `ребенок → ребёнок`, `учет → учёт`, `шел → шёл` — да; а `все`, `мел`, `совершенное` — нет (могут быть и `все`, и `всё`).
Источник: словарь Е. В. Пчёлова (eyo-kernel), методология ёфикации.

#### Верификация и разночтения с типографом А. Лебедева

При разработке русских правил мы прогоняли тестовые тексты через [онлайн-типограф А. Лебедева](https://www.artlebedev.ru/typograf/) и сверяли результаты — это удобный референс с большой базой пользователей. Письменными источниками решений оставались **Ководство** Лебедева, **Справочник** Мильчина, **Справочник** Розенталя и ГОСТы; типограф — инструмент верификации, не норма.

В ряде случаев мы получили расхождения. Ниже — где и почему мы отклонились.

| # | Кейс | Типограф Лебедева | Мы | Обоснование |
|---|---|---|---|---|
| 1 | `см. https://example.com` | NBSP между сокращением и URL | Обычный пробел | URL/email — opaque-токены, типографически не считаются «следующим словом». NBSP к адресу делает его визуально нерасчленимым с предыдущим словом, что нарушает ритм текста. |
| 2 | `или foo@bar.com` | NBSP между проклитикой и email | Обычный пробел | То же соображение, что и для URL. |
| 3 | `1799-1837 гг.` | Обычный дефис между годами | Em-dash + Word Joiner: `1799<WJ>—<WJ>1837<NBSP>гг.` | Лебедев в **Ководстве §161** сам предписывает em-dash без пробелов для диапазонов. Word Joiner добавлен, чтобы Figma (UAX#14) не разрывала строку по em-dash. У типографа Лебедева автозамены `1799-1837 → 1799—1837` не реализовано — видимо, недостаточно надёжно различает диапазон от других случаев. Мы ограничились паттерном `\d{4}-\d{4}` (4+4 цифры) и названиями месяцев — там риск ложных срабатываний минимален. |
| 4 | `и т.д.` | Иногда `т.д.` (слитно) | `т. д.` (NBSP между буквами) | Соответствует **Ководству §95** Лебедева («между буквами в сокращении ставится пробел») и **Мильчину §13.6**. Норма русского языка. |
| 5 | `мы шли`, `ты ведём` | NBSP (pronoun + verb клеятся) | Обычный пробел | Лебедев клеит многобуквенные местоимения с глаголом, но не клеит их с предлогом (`или мы` — без NBSP). Это требует морфологического анализа (различить subject pronoun от object pronoun), что вне scope типографа. Мы выбрали более консервативное и последовательное правило: многобуквенные местоимения всегда свободные. |
| 6 | `или мы` | Обычный пробел | NBSP (`или<NBSP>мы`) | Прямое следствие предыдущего пункта: мы клеим проклитики ко всему, что справа, без морфо-различения. Это последовательнее, чем у Лебедева. |
| 7 | `Г-н Иванов` | Обычный пробел | NBSP (`Г-н<NBSP>Иванов`) с неразрывным дефисом U+2011 внутри | **Мильчин §13.4**: сокращённые обращения и звания пишутся в неразрывном сочетании с фамилией. Сам Лебедев в **Ководстве** это же предписывает. У типографа Лебедева, по-видимому, не реализовано или не срабатывает в начале предложения. |
| 8 | `— это` | NBSP после em-dash | Обычный пробел | Стандартное правило (Розенталь §51, Мильчин): «висячий em-dash» закрывается NBSP **слева** (em-dash не отрывается от предыдущего слова в конце строки). NBSP **справа** делал бы конструкцию `слово — слово` неразрывным чанком — это обедняет вёрстку и плохо смотрится на узких колонках. Лебедев тут агрессивнее общепринятого. |
| 9 | `н. э. —` | Обычный пробел между `э.` и `—` | NBSP (`э.<NBSP>—`) | Лебедев в **Ководстве** сам предписывает NBSP перед em-dash. Его типограф этого правила в данной позиции не выполняет — мы соблюдаем его же письменное правило. |
| 10 | `1234567 единиц` | Без группировки, NBSP между числом и сущ. | `1<NBSP>234<NBSP>567 единиц` (группировка тысяч, без NBSP к сущ.) | Группировка — по **ГОСТ 8.417-2002**, порог ≥5 цифр (4-значные годы вроде `1991` остаются нетронутыми). NBSP к сущ. ставим только для курируемого списка квантификаторов (валюта/время/дата) — иначе `243 голубя` тоже превратится в неразрывное, что неверно. |
| 11 | `ребенок` | Не ёфицирует | `ребёнок` | Реализация ёфикации — наш дизайн-выбор для UI-макетов, где грамотный `ё` повышает читаемость. Используем safe-словарь eyo-kernel (~107 000 словоформ без омонимов) — ложных замен не возникает. |
| 12 | `И/или` (с заглавной в начале предложения) | Не подсвечивает разделитель | `И/или` (без пробелов вокруг косой, заглавная сохраняется) | Устойчивое сочетание, нормализуется как единое целое. Сохранение регистра — техническая аккуратность. |

**Резюме:** там, где Лебедев в **Ководстве** и где **Мильчин/Розенталь** расходятся с поведением его онлайн-типографа, мы следуем письменным источникам, а не имплементации. Там, где Лебедев предлагает менее консервативный подход (склейка местоимений, NBSP после em-dash), мы оставляем поведение, которое:
1. Не требует морфологического анализа.
2. Даёт меньше ложных срабатываний.
3. Создаёт более «дышащую» вёрстку для UI-макетов в Figma.

### 2.3. Английский (`en`)

- **Smart quotes** `"…"` → `“…”`, вложенные `'…'` → `‘…’`.
- **Smart apostrophe** в словах: `don't` → `don’t`, `lovers'` → `lovers’`, `who's` → `who’s`.
- **Primes** для цифр: `12'` → `12′`, `12"` → `12″` (футы/дюймы, минуты/секунды).
- **Honorifics + NBSP** перед именем с заглавной: `Dr. Smith`, `Mr.`, `Mrs.`, `Ms.`, `Prof.`, `St.`, `Sr.`, `Jr.`, `Rev.`, `Capt.`, `Lt.`, `Col.`, `Gen.` В конце предложения сокращение не трогается.
- **Латинские сокращения + NBSP**: `e.g.`, `i.e.`, `etc.`, `vs.` нормализуются с правильными пробелами.
- **Em-dash** (`--` → `—`) между не-цифрами, с обычными пробелами по обе стороны (отличие от русского).
- **Запятая в валютных числах**: `$1234.5` → `$1,234.5` (US-формат). Произвольные числа без валютного префикса не группируются.
- **Артикли/предлоги + NBSP**: `a`, `an`, `the`, `and`, `but`, `or`, `to`, `of`, `in`, `on`, `at`, `by`, `as`, `I` — NBSP справа.

Источник: Chicago Manual of Style.

### 2.4. Французский (`fr`)

- **Гийеметы** `«…»` с **узким** неразрывным пробелом (NNBSP, U+202F) внутри: `« …<NNBSP>»`.
- **NNBSP** перед `!`, `?`, `:`, `;` и закрывающей `»`.
- **Smart liaison apostrophe**: `c'est` → `c’est`, `l'arbre` → `l’arbre`, `aujourd'hui` → `aujourd’hui`.
- **Em-dash** в репликах диалогов с пробелами `—<NBSP>…`.
- **Тысячи NNBSP**: `1 234 567`.

Источник: Lexique des règles typographiques en usage à l’Imprimerie nationale.

---

### 2.5. Немецкий (`de`)

- **Кавычки** `„…“` (нижняя открывающая, верхняя закрывающая).
- **Composite + NBSP внутри**: `z.B.` → `z. B.`, `u.a.` → `u. a.`, `d.h.` → `d. h.`, `u.ä.` → `u. ä.`, `n.Chr.`/`v.Chr.`, плюс `bzw.`, `usw.`, `etc.`
- **Тысячи точкой**: `1.234.567`. Не трогает версии (`1.0.0`) и десятичные с запятой (`3,14`).

Источник: Duden, рекомендации Rat für deutsche Rechtschreibung.

### 2.6. Испанский (`es`)

- **Кавычки** `“…”` по умолчанию; не трогает `«…»`, если уже расставлены (традиция es-AR).
- **Парные `¿…?` и `¡…!`** добавляются, если предложение заканчивается на `?`/`!`, начинается с заглавной и не содержит уже `¿`/`¡`. Только на чистых предложениях — не переписываем риторические фрагменты с серединной пунктуацией.
- Группировка тысяч не выполняется.

Источник: RAE (Real Academia Española), Diccionario panhispánico de dudas.

### 2.7. Прочие языки

| Язык | Особенности |
|---|---|
| **Итальянский (it)** | Кавычки `«…»`. Артикли и предлоги (`il`, `la`, `lo`, `di`, `a`, `da`, `in`, `con`, `su`, `per`, `fra`, `tra`, `e`, `o`) + NBSP. Сокращения `Sig.`, `Dott.`, `Egr.` + NBSP. |
| **Польский (pl)** | Кавычки `„…"`. Однобуквенные предлоги/союзы (`i`, `a`, `o`, `u`, `w`, `z`, плюс `ze`, `we`) + NBSP. Тысячи NBSP. Десятичная запятая (`3,14`). |
| **Португальский (pt)** | Кавычки `“…”`. NBSP перед единицами и валютой `R$`. Тильды (`ã`, `õ`) — детект-маркер. |
| **Голландский (nl)** | NBSP перед единицами. Десятичная запятая. Тысячи точкой (nl-BE) / NBSP (nl-NL). |
| **Украинский (uk)** | Короткие предлоги (`в`, `у`, `з`, `із`, `й`, `та`, `а`, `і`) + NBSP. Сокращения `стор.`, `рис.`, `м.` + узкий NBSP. Кавычки `«…»`. |
| **Сербская кириллица (sr-Cyrl)** | Кавычки `„…"`. Базовые правила NBSP/проклитик от русского без ёфикации (в сербском алфавите нет `ё`). |
| **Босн./хорв./серб. латиница (bcs)** | Кавычки `„…"`. Базовые правила. |

### 2.8. Математика (любой язык, инлайн)

- **Степени**: `x^2` → `x²`, `y^10` → `y¹⁰` (Unicode superscript).
- **Индексы**: `log10` → `log₁₀`, `H_2O` → `H₂O`.
- **Корни**: `sqrt(x)` → `√x`, `sqrt(2)` → `√2`.
- **Греческие буквы** в LaTeX-нотации: `\alpha` → `α`, `\beta` → `β` и т. д. для всего алфавита.
- **Функции**: `sin(x)` → `sin x`, `log(x)` → `log x`, `lim(x→0)` → `limₓ→₀`.
- **Дроби** простые: `1/2` → `½`, `3/4` → `¾` и др. из стандартного набора Unicode.
- **Операторы**: `+-` → `±`, `a*b` → `a · b` (умножение точкой), `inf` → `∞`, `pi` → `π` (строчное — заглавное `Pi` НЕ конвертируется, может быть именем).
- **Стрелки**: `->` → `→`, `<-` → `←`, `=>` → `⇒`, `<=>` → `⇔`, `-->` → `⟶`.

Не трогает LaTeX/MathML-блоки, только короткие инлайн-выражения.

Источник: ISO 80000-2 (математические обозначения), Unicode Math Class table.

### 2.9. Определение языка

Двухэтапный детект:
1. **Уникальные маркеры**: если в тексте есть символ, который встречается только в одном поддерживаемом языке (`ä/ö/ü/ß` → de, `ñ/¿/¡` → es, `ò/ì` → it, `ą/ę/ł` → pl, `ã/õ` → pt, `ç/œ` → fr и т. п.) — этот язык возвращается мгновенно.
2. **Soft-scoring**: общие диакритики (вес 1 за символ) + характерные частотные слова (вес 5 за слово). Побеждает наибольший счёт. На ничьих приоритет примерно по числу носителей: en > es > pt > fr > de > it > pl > nl.

URL и email маскируются **до** детекта, чтобы латиница адресов не перевешивала кириллический текст.

При смешанных скриптах побеждает доминирующий (счёт кириллических vs латинских букв). При равенстве — кириллица.

## 3. Что мы сознательно НЕ делаем

### 3.1. Не клеим многобуквенные местоимения

`мы шли`, `ты ведёшь`, `он сказал` — между ними остаётся обычный пробел.

**Почему:** часть стилистических руководств рекомендует клеить местоимение к глаголу (subject + verb как единый блок), часть — нет. Без морфологического анализатора нельзя надёжно отличить subject pronoun от object pronoun (`он сказал` vs `с ним`) или от вводной конструкции. Универсальное склеивание всех 2+ буквенных местоимений даёт слишком много ложных склеек. Однобуквенное `я` — исключение по правилу «не оставлять однобуквенные слова в конце строки».

### 3.2. Не клеим произвольные конструкции «число + существительное»

`243 голубя`, `100 коробок`, `5 деревьев` — остаются с обычным пробелом.

**Почему:** правило «любое число + любое следующее слово через NBSP» даёт огромное число ложных склеек. Корректно клеить только узкие классы — валюту, время, дату. Эти классы зафиксированы в курируемом списке `QUANTIFIER_NOUNS`. Расширяется точечно, по запросу.

### 3.3. Не разрешаем неоднозначные `ё`

`все` (мн. число от «весь») и `всё` (наречие) пишутся в одинаковых словоформах одинаково. Без контекста (морфоанализ, синтаксис) различить нельзя. Поэтому **не заменяем** `все → всё`, `мел → мёл`, `совершенное → совершённое` и т. п. Заменяются только формы, где альтернативы без `ё` не существует в современном русском.

**Почему:** ложная ёфикация хуже её отсутствия. Лучше пропустить корректную замену, чем поставить ошибочную.

### 3.4. Не расставляем мягкие переносы (U+00AD)

Не вставляем soft hyphens для авто-разбиения длинных слов.

**Почему:** требует словаря слогораздела на каждый язык, плюс соблюдения языковых правил переноса (запрет однобуквенного остатка, запрет переноса аббревиатур и т. п.). Это отдельная задача масштаба собственно «русский переносчик» — выходит за пределы типографа.

### 3.5. Не преобразуем числовые диапазоны кроме годов и месяцев

`10-12 кг`, `15-20`, `2024-12-31` — дефис остаётся ASCII-дефисом.

**Почему:** четырёхзначный паттерн `\d{4}-\d{4}` достаточно надёжно определяется как годы. Произвольные диапазоны типа `10-12` могут быть номерами артикулов, версий ПО (`10.1-12.3`), интервалами дат в ISO-формате, диапазонами страниц. Менять их на en/em-dash рискованно.

### 3.6. Не делаем проверку орфографии

Опечатки в обычных словах не правятся.

**Почему:** out of scope. Это задача spell-checker'а с морфоанализом, словарём и языковой моделью. Типограф работает с формой, не с лексикой.

### 3.7. Не расставляем пробелы после знаков препинания

`привет,мир` остаётся как есть.

**Почему:** редкая проблема, риск false positives высок (внутри URL, артикулов, дробных чисел, имён файлов).

### 3.8. NBSP только перед em-dash, не после

`слово — слово` → `слово<NBSP>— слово` (NBSP слева, обычный пробел справа).

**Почему:** правило «висячий em-dash» закрывается уже одним NBSP слева — em-dash не оторвётся от предыдущего слова в конце строки. NBSP справа делал бы конструкцию `слово — слово` единым неразрывным чанком, что создаёт «дырявые» переносы и обедняет вёрстку.

Исключение: дата-диапазоны (`1991—1995`) — em-dash оборачивается word joiner'ом с обеих сторон, чтобы запретить перенос внутри диапазона.

### 3.9. Не трогаем уже расставленные кавычки

Если в тексте уже стоят `„…"` или `«…»` (правильно или ошибочно для текущего языка), оставляем как есть. Конвертируем только прямые `"…"`.

**Почему:** пользователь мог намеренно использовать вторичный стиль (например, цитата внутри цитаты, авторский выбор). Перекраивать чужие решения — out of scope.

### 3.10. Не делаем эмодзи нормализацию и unification

`😀` остаётся `😀`, ZWJ-семейки (`👨‍👩‍👧`) — целыми.

**Почему:** эмодзи — отдельная проблемная область с собственными правилами variation selectors, skin tones, ZWJ-последовательностями. Гарантируем только их сохранность (не дробим surrogate pairs).

### 3.11. Не редактируем URL и email

Внутренние дефисы, точки, query-string'и сохраняются. Никаких NBSP, smart quotes, минусов внутри адресов.

**Почему:** copy-paste должен возвращать рабочий адрес. Любая «улучшающая» правка превращает URL в нерабочий мусор.

### 3.12. Не определяем язык на коротких фрагментах надёжно

На строках в 1–5 слов детект работает «best effort».

**Почему:** слишком мало сигнала. Например, `OK` или `Hello` могут быть на десяти разных языках. На длинных текстах детект надёжен (>20 символов значимого текста + хотя бы одно характерное слово).

### 3.13. Не обрабатываем диалоги с тире как отдельный случай

`— Привет, — сказал он.` обрабатывается общими правилами тире, без специальной логики «реплика → ровно один пробел после тире».

**Почему:** дизайн-выбор. Диалоги в Figma-макетах встречаются заметно реже, чем буллет-списки и нумерованные пункты. Универсальное правило «тире — со стандартными пробелами» работает и для тех, и для других.

### 3.14. Не различаем em-dash и en-dash в русском

В русском мы используем em-dash (`—`) везде, где нужно тире, включая диапазоны. En-dash (`–`) встречается только если уже стоял в тексте — мы его не вставляем.

**Почему:** в русской традиции en-dash в диапазонах — заимствование из англоязычной типографики, не закреплённое нормативно. ГОСТ и большинство справочников требуют em-dash для всех случаев тире.

## 4. Технические особенности реализации

**Маски** — URL/email/бренды заменяются placeholder'ами равной длины из области U+E000–U+F8FF (Unicode Private Use Area). Длина сохраняется, чтобы не сдвигать индексы при обратной замене. Lookahead'ы правил **не пересекают** PUA — поэтому проклитика/сокращение не клеится к замаскированному адресу через NBSP.

**Идемпотентность** — все правила гарантируют, что повторный прогон не вносит изменений. Property-based тесты на ~12 языков проверяют это автоматически (fast-check).

**Многопроходность** — пайплайн выполняет до 3 итераций (math → common → lang-specific), пока результат не стабилизируется. На практике сходимость за 1–2 прохода.

**Минимальные правки** — LCS-диф выдаёт точечные `(start, end, text)` правки вместо полной перезаписи узла. Figma-стили (bold, italic, цвет, шрифт) сохраняются.

**NBSP-варианты:**
- `U+00A0` (NBSP, обычный неразрывный пробел) — для русских/восточноевропейских проклитик.
- `U+202F` (NNBSP, узкий неразрывный пробел) — для французской пунктуации.
- `U+2060` (WJ, Word Joiner) — там, где нужен zero-width неразрывный связыватель (диапазоны дат).
- `U+2011` (NBH, неразрывный дефис) — внутри `г-н`, `д-р`, `р-н`.
- `U+2212` (MINUS SIGN, юникодный минус) — для отрицательных чисел в финансовом контексте.
