# Typo Graf

[English](README.md) · **Русский**

Плагин для Figma, который приводит текст в макетах к типографическим нормам — расставляет неразрывные пробелы, кавычки, тире, степени, единицы, чинит апострофы и ёфикацию. Одиннадцать языков, защита URL/email/брендов, сохранение стилей, оффлайн.

## Почему он

- **Лёгкий.** Бандл < 1 МБ — вместе со встроенным словарём ёфикации на ~107 000 словоформ. Никаких UI-фреймворков, заставок и онбордингов.
- **Без задержек.** Открывается мгновенно: прогресс-бар и кнопка «Отменить» — больше ничего на экране.
- **Оффлайн.** Все правила, словари и логика — в бандле. `networkAccess: none` в манифесте, ничего не уходит наружу.
- **Сохраняет стили.** `bold`, `italic`, шрифты, размеры и цвета остаются на месте — правки идут через диапазоны, а не перезаписью текста узла.
- **Не трогает URL, email и бренды.** Защищены равной длины маской на время прохода.
- **Один прогон — одно undo.** Все изменения откатываются обычным `Ctrl/⌘+Z`.
- **Тык — и работает.** Без настроек, без диалогов, без выбора режима.
- **Идемпотентен.** Повторный прогон не добавляет дрейфа — лишних правок не делается.

## Как пользоваться

1. **Plugins → Typo Graf** в меню Figma. Если что-то выделено — обрабатывает текстовые узлы внутри выделения (включая вложенные). Если ничего не выделено — пройдётся по всем текстовым узлам текущей страницы.
2. Дождитесь окончания (или нажмите «Отменить»).
3. По завершении плагин показывает сводку: сколько изменений, на скольких узлах, по каким языкам.

Лимит за один запуск — 2000 узлов. Узлы текстом длиннее 5000 символов пропускаются и попадают в итоговую сводку.

## Языки

Определяются автоматически на уровне каждого текстового слоя в два этапа. URL и email маскируются до детекта, чтобы адреса не влияли на счёт:

1. **Уникальные маркеры** — если в тексте есть символ, который встречается только в одном из поддерживаемых языков, этот язык возвращается мгновенно, без подсчёта.
2. **Soft-скоринг** — если уникального маркера нет, плагин складывает два сигнала по каждому языку: общие диакритики (`é`, `à`, `è`, …) с весом 1 за символ и частотные характерные слова (`perché`, `der/die/das`, `het/een`, …) с весом 5 за слово. Побеждает наибольший счёт. На ничьих приоритет примерно по числу носителей в мире: `en` > `es` > `pt` > `fr` > `de` > `it` > `pl` > `nl`.

| Язык | Уникальные маркеры (мгновенно) | Soft-маркеры (скоринг) |
|------|--------------------------------|------------------------|
| `ru` | кириллица (фолбэк) | — |
| `uk` | `і ї є ґ` | — |
| `sr-Cyrl` | `ђ ћ ј љ њ` | — |
| `en` | латиница (фолбэк) | — |
| `de` | `ä ö ü ß` | — |
| `es` | `ñ ¿ ¡` | `á í ó ú ü` |
| `fr` | `ç œ ÿ æ` | `à â è é ê ë î ï ô ù û` |
| `it` | `ò ì` (грав на o/i) | `à è ì ò ù` |
| `pt` | `ã õ` (назальные тильды) | `â ê ô` |
| `pl` | `ą ę ł ń ś ź ż` | `ó` |
| `nl` | частотные слова: `het`, `een`, `van`, `zijn`, `niet`, `maar` | — |
| `bcs` | `č š ž` (`bs`/`hr`/`sr` латиница) | — |

При смешанных скриптах побеждает **доминирующий** (счёт кириллических vs латинских букв). Например, «It is awesome, but мир exists» → `en`, «Привет, hello, как дела?» → `ru`. При ничье — кириллица.

## Правила

### Общие (все языки)

- `...` → `…` (U+2026), лишние пробелы вокруг чистятся.
- `10-12` → `10–12` (en dash для числовых диапазонов).
- Двойные пробелы (в т. ч. после `.!?`) → одинарный.
- Число + единица/валюта/процент → NBSP. Распознаются: SI (кг/г/см/мм/м/км/л/мл), время (мин/сек/ч), частота (Гц/кГц/МГц/ГГц), электричество (Вт/кВт/В/А/Ом), свет/звук (лм/лк/дБ/дБА), давление (Па/кПа/МПа/бар/атм), энергия (Дж/кДж/кал/ккал), цифровые (бит/байт + Кб/Мб/Гб/Тб и старые ГОСТовские МБ/ГБ/ТБ + Мбит/Кбит/Гбит), изображение/UI (Мп/мегапиксель, px, dpi, ppi, fps), валюты (₽/€/$/£/¥/¢/₸/₴/₪), `%`/`‰`.
- ISO 8601 даты (`2024-12-31`, `2024-12-31T23:59:59+03:00`) защищены: дефисы внутри датной части оборачиваются U+2060, чтобы другие правила не сломали их в en-dash, а сама дата осталась неразрывной в вёрстке.
- `12''` → `12″`, `12'` → `12′` (праймы).
- `45 deg` → `45°`.
- `(c)`/`(C)` → `©`, `(r)`/`(R)` → `®`, `(tm)`/`(TM)` → `™` (регистронезависимо).
- CRLF / одиночные CR → LF. На входе — NFC-нормализация (`e` + combining acute → `é`).
- Длинные числа (≥5 цифр) группируются по тысячам с разделителем по локали (см. ниже).

### Математика

Работает на инлайн-выражениях, не лезет в LaTeX/MathML-блоки.

- **Степени:** `x^2 → x²`, `y^10 → y¹⁰`, `10^-3 → 10⁻³`, `e^(x) → eˣ`, `(a+b)^n → (a+b)ⁿ`.
- **Индексы:** `x_1 → x₁`, `log_10(x) → log₁₀ x`.
- **Умножение:** `a*b → a · b`, цепочки `a*b*c → a · b · c`. Жирный (`**...**`) не трогается.
- **Деление:** `a/b → a / b`, `1/2 → ½`, `(a+b)/(c+d)` без замены `/` на `÷`.
- **Сравнения:** `!= → ≠`, `<= → ≤`, `>= → ≥`, `= → =` (с пробелами), пробелы вокруг `≈ ≃`.
- **Знаки:** `+- → ±`, `-+ → ∓`.
- **Стрелки:** `-> → →`, `<- → ←`, `--> → ⟶`, `=> → ⇒`, `<=> → ⇔`.
- **Константы и функции:** строчное `pi → π`, `sqrt(x) → √x`, `inf → ∞`, `\alpha → α` (и остальные греческие), `sin(x) → sin x`, `log10(x) → log₁₀ x`, `lim(x→0) → limₓ→₀`, `vec(a) → a⃗`.
- Заглавное `Pi` **не** конвертится (может быть именем собственным или началом предложения).
- Em-dashes (`—`) между не-цифрами сохраняются как типографические; только ASCII `-`/`–` между алфавитно-цифровыми токенами превращаются в математический минус `−` (U+2212).

### Русский (`ru`)

- NBSP после **проклитик** — коротких предлогов, союзов и forward-частиц: `в дом`, `на улице`, `и т. п.`, `даже он`. Однобуквенное местоимение `я` входит сюда же (однобуквенные слова не должны висеть в конце строки) — `Я закончил` → `Я<NBSP>закончил`. Остальные местоимения остаются свободными.
- NBSP перед **энклитиками** `бы`, `ли`, `же`, `ль` — они тянутся **только к слову слева**. Пример: `всё было бы иначе` → `всё было<NBSP>бы иначе` (слово `иначе` свободно переносится). Списки проклитик и энклитик не пересекаются — это инвариант, зафиксированный в [src/lib/ruLib.ts](src/lib/ruLib.ts).
- Инициалы: `А. С. Пушкин` — NBSP между инициалами и фамилией.
- Аббревиатуры с точкой: `г. Москва`, `ул. Ленина`, `№ 8`, `§ 104`, `1981 г.` — NBSP после.
- Закрывающие сокращения «годами»/«веками» (`гг.`, `вв.`) — NBSP только **перед** (чтобы `1991<NBSP>гг.` не разрывалось), после — обычный пробел: это замыкающие токены, перенос строки перед следующим словом допустим.
- Составные сокращения: `и т. д.`, `т. е.`, `до н. э.` — нормализованы вместе с NBSP. Хвост composite (буква `д.` в `и т. д.`) **не** прилипает к следующему слову, поэтому `и т. д. А. С. Пушкин` оставляет обычный пробел между `д.` и инициалами.
- Дефисные сокращения без точки: `г-н`, `г-жа`, `д-р`, `р-н` — неразрывный дефис (U+2011) внутри и NBSP после.
- Кавычки-ёлочки `«…»`. Пунктуация прижимается к закрывающей.
- Тире: двойной дефис `--` и одиночный `-` между не-цифровыми токенами → `—`, схема `слово<NBSP>—<space>слово`.
- Диапазоны дат с em-dash без пробелов, обёрнутые в U+2060 (word joiner) — чтобы диапазон оставался неразрывным целиком: `1991-1995` → `1991⁠—⁠1995`, `январь-март` → `январь⁠—⁠март`. Без word joiner Figma по UAX#14 может разорвать строку прямо по em-dash.
- Число + квантификаторное существительное (валюта/время/дата) → NBSP: `5 рублей`, `12 января`, `300 минут`, `1991 году`. Список узкий и кураторский — обычные счётные сущ. вроде `243 голубя` **не** клеятся.
- Юникодный минус `−` (U+2212) для отрицательных финансовых значений: `-300 ₽` → `−300 ₽`, `-15 %` → `−15 %`. Дефисы в буллетах не трогаются.
- `и / или` → `и/или` (без пробелов вокруг косой). В начале предложения сохраняем заглавную: `И / или …` → `И/или …`.
- Группировка тысяч NBSP для чисел ≥5 цифр: `1234567` → `1 234 567`. Годы (`1991`) и 4-значные идентификаторы остаются.
- **Ёфикация на ~107 000 словоформ** из словаря [eyo-kernel](https://github.com/e2yo/eyo-kernel). Заменяются только «безопасные» формы — те, где нет омонима без `ё`. То есть `ребенок → ребёнок`, `учет → учёт`, `шел → шёл`, но `все` остаётся как есть (омоним мн. ч. от «весь»). Случайных правок не делаем.

### Английский (`en`)

- Smart quotes: `"..."` → `“…”`, вложенные `'...'` → `‘…’`.
- Апострофы в словах и на конце: `don't` → `don’t`, `lovers'` → `lovers’`.
- Праймы: `12''` → `12″`, `12'` → `12′`.
- `--` → `—` без навязывания пробелов вокруг (по исходному стилю).
- Диапазоны: `10-12 km` → `10–12 km`.
- Валюты: `$1234.5` → `$1,234.5` (US-формат). Порядок `$ 300` / `300 $` сохраняется, NBSP добавляется.
- NBSP после служебных слов (`a`, `an`, `the`, `and`, `but`, `or`, `to`, `of`, `in`, `on`, `at`, `by`, `as`, `I`).
- NBSP перед единицами: `15 km`, `20 %`, `60 mph`, `5 dB`, `100 MHz`.
- Латинские сокращения: `e.g.` → `e. g.`, `i.e.` → `i. e.` (NBSP внутри), затем NBSP между сокращением и следующим словом: `e.g. for example` → `e. g. for example`. Аналогично `etc.` и `vs.`.
- Honorifics + NBSP: `Dr. Smith` → `Dr. Smith`, также `Mr.`, `Mrs.`, `Ms.`, `Prof.`, `St.`, `Sr.`, `Jr.`, `Rev.`, `Capt.`, `Lt.`, `Col.`, `Gen.` — только перед именем с заглавной. В конце предложения сокращение остаётся как есть.

### Французский (`fr`)

- Гийеметы `« … »` с узкими NBSP (U+202F) внутри.
- Узкий NBSP перед `; : ? ! »`.
- Число + единица/валюта/процент → узкий NBSP.
- Smart apostrophe для liaison: `l'arbre` → `l’arbre`, `d'accord` → `d’accord`, `qu'on` → `qu’on`, `Aujourd'hui` → `Aujourd’hui`.
- Em-dash из `--` и из `-` между не-цифровыми словами, с **обычными** пробелами по обе стороны (в отличие от русского).
- Группировка тысяч узким NBSP.

### Украинский (`uk`)

- Кавычки `«…»`.
- NBSP после коротких предлогов: `в`, `у`, `з`, `із`, `й`, `та`, `а`, `і`.
- Узкий NBSP после `№`, `§`, `стор.`, `рис.`, `м.`.
- Число + единица/валюта/процент → узкий NBSP.
- Группировка тысяч узким NBSP.

### Итальянский (`it`)

- Гийеметы `«…»` (по умолчанию; оставляет `“…”`, если уже встречены).
- NBSP после артиклей (`il`, `la`, `lo`, `i`, `gli`, `le`, `un`, `una`, `uno`) и коротких предлогов/союзов (`di`, `a`, `da`, `in`, `con`, `su`, `per`, `fra`, `tra`, `e`, `o`, `ma`, `se`).
- Сокращения с точкой + NBSP перед именем с заглавной: `Sig. Rossi`, `Dott. Verdi`, `Prof. Bianchi` и т. п.
- NBSP перед единицами/валютой/процентом.
- Группировка тысяч NBSP.

### Польский (`pl`)

- Кавычки `„…”`.
- NBSP после однобуквенных предлогов/союзов: `i`, `a`, `o`, `u`, `w`, `z`, плюс фонетические `ze`, `we`.
- NBSP перед единицами, валютой, процентом (включая `zł`).
- Десятичная запятая (3,14) не трогается.
- Группировка тысяч NBSP.

### Португальский (`pt`)

- Smart quotes `“…”` (по умолчанию; оставляет `«…»`, если уже встречены — традиция PT-PT).
- NBSP перед единицами, валютой (`R$`, `$`, `€`), процентом.
- Группировка тысяч NBSP.

### Голландский (`nl`)

- Кавычки `„…”`.
- NBSP перед единицами, валютой, процентом.
- Десятичная запятая не трогается.
- Группировка тысяч NBSP.
- Детект — по частотным словам (`het`, `een`, `van`, `zijn`, …): голландский редко использует диакритики. На коротких неоднозначных строках возможны промахи — в таких случаях полезно задать язык слоя в Figma вручную.

### Немецкий (`de`)

- Кавычки `„…“`. Если в тексте уже стоит `»…«` — не трогает.
- NBSP перед единицами и валютой.
- Десятичная запятая не трогается.
- Группировка тысяч **точкой**: `1234567` → `1.234.567`. Версии (`1.0.0`) и десятичные (`3,14`) пропускаются, потому что уже содержат точки/запятые.
- Составные сокращения: `z.B.` → `z. B.`, `u.a.` → `u. a.`, `d.h.` → `d. h.`, `u.ä.` → `u. ä.`, `n.Chr.`/`v.Chr.` — NBSP внутри, затем NBSP перед следующим словом. Также `bzw.`, `usw.`, `etc.` получают NBSP после.

### Испанский (`es`)

- Кавычки `“…”` по умолчанию. Если уже стоит `«…»` — не трогает.
- NBSP перед единицами и валютой.
- Парные `¿…?` и `¡…!`: если предложение заканчивается на `?`/`!`, начинается с заглавной и не содержит `¿`/`¡`, открывающий знак добавляется. Срабатывает только на чистых предложениях, чтобы случайно не переписать риторические фрагменты с серединной пунктуацией.

### BCS (`bs`/`hr`/`sr` латиница)

- Кавычки нормализуются по встреченному варианту (`«…»` или `„…“`). По умолчанию — `„…“`.
- NBSP перед единицами и валютой.

### Сербская кириллица (`sr-Cyrl`)

- Те же правила, что и у русского (инициалы, короткие предлоги, em-dash, единицы, …), **кроме** кавычек (`„…“` вместо `«…»`) и **без** ёфикации (в сербской кириллице нет буквы `ё`).

## Что плагин НЕ делает

- **Не трогает URL и email.** Маскируются равной длины placeholder'ами, после прохода восстанавливаются обратно. Проклитики и сокращения перед URL/email **не** прилипают к ним: `см. https://...` и `на foo@bar.com` остаются с обычным пробелом — адрес как opaque-токен, copy-paste возвращает валидный адрес.
- **Не ломает переносы строк.** `\n` сохраняется, склейка через перенос не делается.
- **Не лезет в LaTeX/MathML-блоки.** Только короткие inline-операнды.
- **Не редактирует символьное выделение внутри узла.** Только текстовые узлы целиком.
- **Не ёфицирует слова с омонимами.** Чтобы не превратить «все» в «всё» там, где имеется в виду «весь».
- **Не трогает бренды и продуктовые имена.** Токены из `src/dict/brands.txt` (iPhone, JavaScript, Booking.com, Figma и т. п.) защищены. Регистрозависимо: `iPhone` сохраняется, а `iphone` обрабатывается как обычный текст. Свои бренды добавляются правкой файла и пересборкой.
- **Не подключает кастомные шрифты.** Узлы с шрифтом, недоступным в Figma, пропускаются и попадают в итоговый счётчик.

## Интерфейс

Окно 320×160 px:

- Прогресс-бар.
- Текущая операция (`Сканирование…`, `Обработка…`, `Отмена…`).
- Счётчик обработанных узлов: `X / Y`.
- Кнопка «Отменить» — останавливает после текущего узла; всё, что уже применилось, остаётся (откатывается общим `Ctrl/⌘+Z`).

Финальная сводка локализована под язык Figma:

```
Готово. Языки: русский, английский. Изменений: 142. Затронуто узлов: 38.
Ctrl/⌘+Z — отменить
```

## Установка (для разработки)

```bash
npm install
npm run build       # одноразовая сборка
npm run dev         # сборка в watch-режиме
```

Затем в Figma desktop → **Plugins → Development → Import plugin from manifest** → выбрать `manifest.json` в корне репозитория.

```bash
npm run bench       # baseline-производительность
```

После `npm test` подробная диагностика лежит в `test-results/report.json`: по каждому правилу — входной текст, ожидаемый/полученный результат и точные позиции расхождений с код-пойнтами Unicode. `npm run showcase` печатает наглядные before → after по корпусу. `npm run bench` показывает пропускную способность по языкам и длине строки — полезно ловить регрессии при правке правил.

CI (GitHub Actions) гоняет полный набор на каждый push и pull request, а `test-results/report.json` публикуется как артефакт.

## Ограничения

- Селекция работает на уровне узлов целиком, а не отдельных символов внутри узла.
- Лимит — 2000 текстовых узлов за один запуск; узлы длиннее 5000 символов пропускаются.
- Узлы с недоступными в Figma шрифтами пропускаются.
- Откат — нативным `Ctrl/⌘+Z` (общая undo-группа на весь прогон).
- Текст со смешанными скриптами обрабатывается по правилам доминирующего скрипта. Для коротких смешанных строк детект — лучшее предположение; URL и email из счёта исключены, поэтому они не сдвигают результат.
